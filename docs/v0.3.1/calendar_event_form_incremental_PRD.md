# 统一日历事件表单｜增量 PRD

## 0. 文档定位

本文档是《知识库日历功能 PRD》的增量补充，仅描述“统一日历事件表单”相关的新增与变更。

除本文明确修改的规则外，原 PRD 中的以下内容保持不变：

- Calendar 独立页面
- 月视图与列表视图
- 日期识别与推荐规则
- Calendar CRUD
- 知识条目与日历事件的关联关系
- 删除、权限、防重复和数据隔离规则
- Dashboard 不展示完整日历

本文档不重复定义上述功能。

---

## 1. 变更目标

将以下三个入口统一为同一个事件表单组件：

1. 知识详情页中的“添加到日历”
2. Calendar 页面中的“新建事件”
3. Calendar 页面中已有事件的“编辑事件”

统一后，三个入口共享相同的字段、校验、提交逻辑和视觉结构，仅默认值不同。

---

## 2. 本次明确变更

### 2.1 统一使用一个表单

新增统一组件：

```tsx
<CalendarEventForm />
```

该组件支持：

```text
create
edit
```

并根据入口区分默认值：

```text
knowledge-detail
calendar-create
calendar-day
calendar-edit
```

---

### 2.2 表单字段

表单仅包含以下字段：

| 字段 | 必填 | 说明 |
|---|---:|---|
| 标题 | 是 | 事件名称 |
| 日期 | 是 | 仅精确到日期 |
| 关联内容 | 否 | 关联原知识条目 |
| 备注 | 否 | 普通多行文本 |

本次明确不加入：

- 事件类型
- 开始时间
- 结束时间
- 提醒
- 重复规则
- 优先级
- 状态
- 地点
- 参与人
- 子任务

---

## 3. 默认值规则

## 3.1 从知识详情页进入

用户点击“添加到日历”后：

- 标题默认使用当前知识条目的标题
- 日期默认使用系统已识别出的推荐日期
- 关联内容默认使用当前知识条目
- 备注默认为空

用户可以修改标题和日期。

### 无推荐日期时

原 PRD 中“日期为空，由用户手动选择”的规则，更新为：

> 如果没有识别到有效推荐日期，日期默认使用今天。

同时在日期字段下方显示：

```text
未识别到明确日期，已默认选择今天。
```

该规则覆盖原 PRD 中与之冲突的描述。

---

## 3.2 从 Calendar 顶部“新建事件”进入

默认值：

- 标题为空
- 日期默认今天
- 关联内容为空
- 备注为空

---

## 3.3 从 Calendar 日期格进入

用户点击月历中的某一天新建事件时：

- 标题为空
- 日期默认使用所点击的日期
- 关联内容为空
- 备注为空

---

## 3.4 编辑已有事件

打开已有事件时：

- 标题回填已有标题
- 日期回填已有日期
- 关联内容回填已有来源
- 备注回填已有备注

编辑模式复用同一个表单，不单独创建另一套编辑界面。

---

## 4. 表单结构

### 4.1 创建模式

```text
新建事件 / 添加到日历

标题 *
[请输入标题]

日期 *
[YYYY-MM-DD]

推荐日期说明
[仅从知识详情页进入且存在推荐信息时显示]

关联内容
[有来源时显示知识标题]

备注
[添加备注……]

[取消] [保存]
```

### 4.2 编辑模式

```text
编辑事件

标题 *
[已有标题]

日期 *
[已有日期]

关联内容
[已有来源]

备注
[已有备注]

[删除事件] [取消] [保存]
```

---

## 5. 标题字段规则

- 必填
- 从知识详情页进入时默认使用知识标题
- 从 Calendar 新建时为空
- 用户可以修改
- 提交前自动去除首尾空格
- 去除空格后不能为空
- 建议最大长度为 120 个字符

本次不额外使用 AI 或正文上下文生成事件名称。

---

## 6. 日期字段规则

- 必填
- 日期格式统一为 `YYYY-MM-DD`
- 只记录日期，不记录小时和分钟
- 从知识详情页进入时优先使用推荐日期
- 无推荐日期时默认今天
- Calendar 顶部新建时默认今天
- 点击日期格新建时默认所点击日期
- 编辑时使用已保存日期
- 用户手动修改后的日期优先于系统默认值

本文档不修改原 PRD 已定义的日期识别算法、候选日期排序和置信度规则。

---

## 7. 推荐日期说明

仅在从知识详情页进入时显示。

存在推荐日期时，显示：

```text
推荐日期：2026-08-01
识别依据：“报名将在 8 月 1 日截止”
```

模糊日期时显示：

```text
该日期由模糊时间推算，请确认。
```

无推荐日期时显示：

```text
未识别到明确日期，已默认选择今天。
```

推荐说明仅用于辅助用户确认，不作为用户必填字段。

---

## 8. 关联内容规则

### 从知识详情页进入

- 自动关联当前知识条目
- 显示知识标题
- 支持查看原内容
- 支持移除关联

### 从 Calendar 页面新建

本次增量不新增知识库搜索器。

默认创建无关联事件。

后续若增加“搜索并关联知识内容”，应作为独立需求处理。

### 编辑事件

- 有关联时显示来源标题
- 支持跳转原知识详情页
- 支持移除关联
- 移除关联不会删除知识内容

---

## 9. 备注字段规则

- 选填
- 使用普通多行文本框
- 不使用富文本编辑器
- 建议最大长度为 2,000 个字符

---

## 10. 提交与状态

### 创建模式

按钮：

```text
取消
保存
```

从知识详情页进入时，主按钮文案也可以显示：

```text
添加
```

### 编辑模式

按钮：

```text
删除事件
取消
保存
```

### 提交规则

- 标题和日期通过校验后才能提交
- 提交过程中禁用保存按钮
- 防止连续点击造成重复创建
- 提交失败时保留用户输入
- 创建成功后关闭表单并刷新页面状态
- 编辑成功后刷新 Calendar 与详情页状态

---

## 11. 与原 PRD 的关系

### 本文覆盖的旧规则

| 原规则 | 新规则 |
|---|---|
| 无推荐日期时日期为空 | 无推荐日期时默认今天 |
| 添加、创建、编辑可分别实现 | 三个入口统一使用同一表单 |
| 表单字段尚未完全固定 | 固定为标题、日期、关联内容、备注 |
| 可考虑事件类型 | 明确不加入事件类型 |

### 保持不变的规则

- Calendar 独立页面
- 不放入 Dashboard
- 日期只精确到日
- 用户确认后才创建事件
- 推荐日期可以被用户修改
- 已添加事件再次点击进入编辑
- 删除事件不删除知识内容
- 同一知识条目默认只关联一个主要事件
- 日期识别、排序和置信度规则不变

---

## 12. 验收标准

### 统一表单

- [ ] “添加到日历”“新建事件”“编辑事件”使用同一组件
- [ ] 表单只包含标题、日期、关联内容、备注
- [ ] 表单不包含事件类型
- [ ] 创建和编辑使用同一套校验与提交逻辑

### 默认值

- [ ] 知识详情页进入时标题默认使用知识标题
- [ ] 知识详情页进入时日期默认使用推荐日期
- [ ] 无推荐日期时默认今天
- [ ] Calendar 顶部新建时日期默认今天
- [ ] 点击日期格新建时日期默认所点击日期
- [ ] 编辑时正确回填已有数据

### 用户操作

- [ ] 用户可以修改标题
- [ ] 用户可以修改日期
- [ ] 用户可以填写备注
- [ ] 有来源时可以查看或移除关联
- [ ] 提交失败时不丢失表单内容
- [ ] 不会因连续点击重复创建事件

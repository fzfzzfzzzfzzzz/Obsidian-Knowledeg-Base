## 1. 当前问题诊断（基于代码审计）

我通读了 `base.html`、`style.css`、`app.js`、`index.html`、`kb_web.py`，定位到以下具体问题：

### 1.1 移动端布局缺陷（最紧急）
- **卡片横向溢出**：`style.css:88` 的 `.card-grid { grid-template-columns: repeat(auto-fill, minmax(320px, 1fr)); }` 最小列宽 320px，叠加移动端容器 `padding: 16px*2`，在 320–360px 视口下总宽超过视口，产生横向滚动条。
- **顶部导航无折叠**：`base.html:12-21` 有 8 个顶部链接（仪表盘/搜索/全部文章/投稿/最近阅读/收藏夹/Ideas/Todos），移动端无汉堡菜单或底部栏，`@media (max-width:640px)` 仅调了 padding/gap（`style.css:345-350`），导航会换行挤压、难以点击。
- **无触摸目标规范**：图标按钮 `.icon-btn`（`style.css:402-411`）`padding:3px 7px; font-size:14px` 实际高度约 24px，远低于 44px 触摸标准；`.card-checkbox` 18px 同样过小。

### 1.2 设计令牌缺失 · 一致性差
- 颜色大量硬编码重复：`#2563eb`（主蓝）、`#8b5cf6`（紫）、`#10b981`（绿）、`#f59e0b`（琥珀）、`#ef4444`（红）散落全文件，无统一 token，改主题/换色需逐处搜索。
- **无暗色模式**：全部使用浅色固定值，夜间使用刺眼。
- 字体、间距、圆角、阴影均无 token 体系，靠散值拼凑。
- 中英混排不统一：导航用 `Ideas / Todos / someday`，中文用户认知负担。

### 1.3 可访问性（WCAG AA 差距）
- 图标按钮无 `aria-label`（`app.js` `renderArticleCard` 中的 📖/⭐/🗑 仅靠 `title`，屏幕阅读器与键盘用户不友好）。
- 全局**未定义 `:focus-visible`**，键盘 Tab 焦点不可见。
- `.btn:disabled { opacity: 0.4 }`（`style.css:234`）仅靠透明度表达禁用，非文本对比不足 3:1。
- 状态徽章主要靠颜色区分（灰=待审、绿=已接受），未辅以图标/文字形状。
- `progress-bar` 渐变、卡片 hover 位移等动效未尊重 `prefers-reduced-motion`。

### 1.4 交互与体验
- **仪表盘无限长**：`index.html` 三区块（未读/已读/稍后读）直接罗列全部卡片，文章多（如 100+）时页面极长，无分页 / 收起 / "显示更多"。
- **加载态简陋**：仅文本"加载中…"，无骨架屏，感知性能差。
- **危险操作体验差**：删除用原生 `confirm()` 两次（`app.js` `deleteArticle` / `batchAction`），不可定制、移动端丑、可访问性弱。
- 批量栏 `top:56px` 硬编码（`style.css:595`），依赖导航高度，导航一改即错位。
- 投稿页 / 搜索页 / 详情页的部分细节未逐一深读，已计入实施分期的风险项。

### 1.5 视觉语言
- emoji 与文字混排（📚 🔍 ⭐ 🗑）风格不统一，缺乏图标系统性。
- 卡片 hover `translateY(-2px)` + 阴影在触屏无意义且耗性能，未用 `@media (hover:hover)` 隔离。
- 整体缺乏清晰的视觉层级节奏（标题/正文/次要信息对比不足）。

## 2. 设计思路概述

**核心原则**：一致性 → 可用性 → 可访问性 → 克制（简洁现代）。不追求炫技，优先让你"看得清、点得准、找得到"。

1. **引入设计令牌系统**：用 CSS 变量集中管理 color / 字体 / 间距 / 圆角 / 阴影 / 动效，一处定义、全局复用，为暗色模式与主题化打底。
2. **双主题（浅色默认 + 暗色）**：基于 `prefers-color-scheme` 自动，并提供手动切换按钮，状态存 `localStorage`。
3. **移动优先响应式**：断点 `≤640（手机）/ 641–1024（平板）/ ≥1025（桌面）`；容器 `max-width` 1200px；修复卡片最小列宽溢出。
4. **导航重构**：桌面采用分组侧边栏（或顶部分组），移动端采用底部标签栏 + 汉堡"更多"，消除换行与溢出。
5. **组件库化**：统一 Button / Card / Tag·Badge / Input·Select / Modal / Toast / Skeleton / EmptyState，每组件含默认/悬停/聚焦/禁用/加载状态。
6. **可访问性基线**：键盘可达、焦点可见、必要 ARIA、`focus-visible`、对比度 ≥4.5:1（大文本 ≥3:1）、触摸目标 ≥44px、`prefers-reduced-motion` 兜底。
7. **性能意识**：用骨架屏替代 spinner；触屏禁用 hover 位移；CSS 变量降低重绘成本。

## 4. 页面结构规划

### 4.1 信息架构（IA）

```
Obsidian KB
├─ 阅读（主场景）
│   ├─ 仪表盘 Dashboard    （统计卡 + 未读/已读/稍后读 标签页）
│   ├─ 全部文章 Articles   （可筛选列表）
│   ├─ 文章详情 Summary    （Markdown 渲染 + 操作栏）
│   ├─ 最近阅读 Recent
│   └─ 收藏夹 Favorites
├─ 创作 / 收集
│   ├─ 投稿 Submit         （粘贴 → ingest → 可选自动总结）
│   └─ 搜索 Search         （关键词 + 多维筛选）
└─ 审阅（AI 建议闭环）
    ├─ Ideas 建议          （候选卡片 → 接受/拒绝 → 写回）
    └─ Todos 建议
```

### 4.2 导航方案（推荐：桌面侧栏 + 移动底栏）
- **桌面（≥1024）**：左侧固定侧边栏（宽 220–240px），顶部 Logo + 分组导航（阅读 / 创作 / 审阅），主内容区右侧留白；主题切换在侧栏底部。
- **平板（641–1024）**：顶部精简条 + 可折叠侧栏（汉堡展开为抽屉）。
- **移动（≤640）**：顶部条（标题 + 主题切换）+ **底部标签栏**（首页/文章/搜索/投稿/更多），"更多"展开抽屉放 最近/收藏/Ideas/Todos。消除导航换行与溢出。

### 4.3 各页面布局要点
| 页面 | 布局 | 关键元素 | 响应式行为 |
|------|------|----------|-----------|
| 仪表盘 | 统计卡(4) + 进度条 + 标签页(未读/已读/稍后读) | 标签页切换、每页"显示更多"、骨架屏 | 手机：统计卡 2 列；标签页横向滚动 |
| 文章详情 | 返回链 + 元信息条 + 操作栏 + Markdown 正文 | 稍后读/收藏切换、删除、重新生成 | 手机：操作栏换行；正文内边距收窄 |
| 全部文章 | 筛选条(状态/类型/收藏) + 卡片网格 | 多选、批量栏 | 同仪表盘网格 |
| 投稿 | 多文本框动态增删 + 自动总结勾选 + 结果区 | 提交、结果卡片、日志折叠 | 手机：单栏堆叠 |
| 搜索 | 搜索框 + 筛选行(类型/状态/标签) + 结果网格 | 实时/回车搜索、结果计数 | 手机：筛选行换行 |
| 最近/收藏 | 卡片网格 + 空态 | 同文章卡 | 同 |
| Ideas/Todos | 候选卡(状态徽章+字段表+操作) | 接受/拒绝按钮、写回反馈 | 手机：单栏 |

## 5. 主要交互流程说明

1. **阅读流程**：列表/卡片 → 点击进入详情 → 自动标记已读（后端已做）→ 详情页可「稍后读 / 收藏」切换（乐观更新，不整页刷新）→ 返回列表。
2. **投稿流程**：投稿页粘贴若干文本/URL → 提交 → 后端写 inbox + ingest →（勾选）自动生成 summary → 结果区显示新增条目 + 日志 → 可跳转详情或继续投稿。
3. **Idea/Todo 审阅流程**：审阅页加载候选 → 对 `pending_review` 卡片点「接受·科研/效率」或「本周/本月/someday」/「拒绝」→ 二次确认 Modal → 写回 Markdown（仅改 status 行 + 备份）→ 卡片状态刷新。
4. **批量操作流**：卡片多选（≥44px 触摸框）→ 顶部/底部批量栏显示已选数 → 选操作（收藏/标签/生成总结/删除）→ 确认 Modal（危险操作）→ Toast 汇总成功/跳过/失败。
5. **搜索/筛选流**：输入关键词或调整筛选 → 调用 `/api/search` → 渲染结果网格 + 计数；空结果给引导空态。
6. **全局态**：主题切换（持久化）、加载骨架屏、错误边界提示、`prefers-reduced-motion` 关闭位移动画。

## 6. 关键界面元素说明（组件规格）

| 组件 | 规格要点 |
|------|----------|
| Button | 变体 primary/secondary/ghost/danger；尺寸 sm(32h)/md(40h)；`min-height` 触屏 44px；`:focus-visible` 焦点环；禁用态加斜纹/图标而非纯透明 |
| Card | 基础卡 + 文章卡变体；hover 位移仅 `@media (hover:hover)`；圆角 `--r-md`；边框 + 轻阴影 |
| Tag / Badge | 类型标签（来源/领域）+ 状态徽章（**颜色 + 文字 + 图标**三要素，不靠颜色 alone）|
| Input / Select | 统一焦点环 `--c-focus`；标签关联 `for`/`id`；错误态红色边框 + 文案 |
| Modal | 替代原生 `confirm`：标题/正文/确认·取消；焦点陷阱；Esc 关闭；危险操作红色强调 |
| Toast | 替代 `alert()`：右下角自动消失，成功/警告/错误三态 |
| Skeleton | 卡片占位脉冲动画（尊重 reduced-motion），替代"加载中…" |
| EmptyState | 统一插画占位 + 一句引导 + 主操作按钮 |
| 导航 | 侧栏(桌面)/底栏(移动)；当前项高对比高亮；ARIA `aria-current` |

## 7. 实施分期建议（确认后执行）

- **Phase A（低风险，纯样式/令牌）**：落地设计令牌、暗色模式、`:focus-visible`、触摸目标 ≥44px、骨架屏、EmptyState、Modal/Toast 基础组件、修复卡片移动端溢出。
- **Phase B（结构）**：导航重构（侧栏 + 移动底栏/抽屉）、仪表盘标签页化 + "显示更多"、批量栏定位改用 CSS 变量/粘性而非硬编码。
- **Phase C（打磨）**：投稿/搜索/详情页细节统一、删除等危险操作接入 Modal、reduced-motion 兜底、全站走查。
- **验证**：本地 `python scripts/kb.py serve`，桌面 + 移动模拟器（375/768/1280）逐页走查，对照 WCAG AA 抽查对比度与键盘可达。
